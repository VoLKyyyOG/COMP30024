<h2><a id="user-content-week-2-problem-solving-and-search" class="anchor" aria-hidden="true" href="#week-2-problem-solving-and-search"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Week 2: Problem Solving and Search</h2>
<h5><a id="user-content-formulate-single-state-search-problem" class="anchor" aria-hidden="true" href="#formulate-single-state-search-problem"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Formulate single state search problem</h5>
<ul>
<li><strong>Formulate a Problem</strong>:
<ul>

<li>Consider the <strong>operations we can do to change state</strong></li>

<ul>
<li>States: Southbank, Unimelb</li>
<li>Actions: Drive</li>
</ul>
</li>
</ul>
</li>
<li>A <strong>solution</strong> is considered a <strong>sequence of actions</strong> leading from the <strong>initial state</strong> to the <strong>goal state</strong>

</ul>
</li>
</ul>
<h5><a id="user-content-analyse-complexity-of-a-search-strategy" class="anchor" aria-hidden="true" href="#analyse-complexity-of-a-search-strategy"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Analyse complexity of a search strategy</h5>
<h5><a id="user-content-apply-a-search-strategy-to-solve-problem" class="anchor" aria-hidden="true" href="#apply-a-search-strategy-to-solve-problem"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Apply a search strategy to solve problem</h5>
<ul>
<li><strong>BFS</strong>
<ul>
<li>Expand breadth wise</li>
<li><strong>Complete if branching factor is finite</strong></li>
<li><em>Time:</em> <strong>O(b<sup>d</sup>)</strong></li>
<li><em>Space:</em> <strong>O(b<sup>d</sup>)</strong></li>
<li><strong>Optimal Path if path cost is uniform, otherwise <em>not complete or optimal in general</em></strong></li>
</ul>
</li>
<li><strong>DFS</strong>
<ul>
<li>Expand depth wise</li>
<li><strong>Not Complete if in Infinite Space</strong></li>
<li><em>Time:</em> <strong>O(b<sup>m</sup>)</strong></li>
<li><em>Space:</em> <strong>O(bm)</strong></li>
<li><strong>Not Optimal Path</strong></li>
</ul>
</li>
<li><strong>Uniform Cost Search</strong>
<ul>
<li>Expand least-cost node</li>
<li><strong>Complete if step size &gt; 0</strong></li>
<li><em>Time:</em> <strong>n nodes g cost &lt;= optimal cost</strong></li>
<li><em>Space:</em> <strong>n nodes g cost &lt;= optimal cost</strong></li>
<li><strong>Optimal</strong></li>
</ul>
</li>
<li><strong>Depth Limited Search</strong>
<ul>
<li>Do depth first to a certain limit</li>
<li><strong>Not Complete</strong></li>
<li><em>Time:</em> <strong>O(b<sup>l</sup>)</strong></li>
<li><em>Space:</em> <strong>O(bl)</strong></li>
<li><strong>Not Optimal</strong></li>
</ul>
</li>
<li><strong>Iterative Deepening Search</strong>
<ul>
<li>Do DFS for each 'step'</li>
<li><strong>Complete</strong></li>
<li><em>Time:</em> <strong>O(b<sup>d</sup>)</strong></li>
<li><em>Space:</em> <strong>O(bd)</strong></li>
<li><strong>Optimal if step size = 1</strong></li>
</ul>
</li>
<li><strong>Bidirectional Search</strong>
<ul>
<li>Search from both goal and start</li>
<li><strong>Complete</strong></li>

<ul>

<ul>

<ul>

<ul>
<li>Whenever you <strong>assign a variable</strong>, <strong>remove this value from any neighbours</strong>
<ul>
<li>e.g. for the australia state colour problem:
<ul>
<li>We assign <em>RED to WA</em>:</li>

</ul>
</li>
<li>Complexity <em>depends on whatever order you check variables/values</em></li>

</ul>

</ul>
</li>
<li><strong>Arc Consistency (AC-3)</strong>
<ul>
<li><strong>Start from a queue of arcs</strong> and <strong>check each node at each arc</strong> whether they share any <strong>values/domain in common</strong>

<li>This <em>checks for conflicts</em></li>
<li>You should <em>do this before backtracking search</em></li>
</ul>
</li>

<h5><a id="user-content-use-conditional-independence-to-simplify-probability-calculations" class="anchor" aria-hidden="true" href="#use-conditional-independence-to-simplify-probability-calculations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Use conditional independence to simplify probability calculations</h5>
<ul>
<li><strong>A and B are independent</strong> iff
<ul>
<li><code>P(A|B) = P(A) or P(B|A) = P(B) or P(A, B) = P(A)P(B)</code></li>
<li>Also means <code>P(A|!B) = P(A) and etc.</code></li>
</ul>
</li>
<li>This means <strong>if <em>A and B are independent</em></strong> then we can <strong>simplify P(A|B,C)</strong>
<ul>
<li>Since <code>P(A|B) = P(A)</code> then <strong><code>P(A|B,C)</code> becomes <code>P(A|C)</code></strong></li>
<li>Also <code>P(A, B|C) = P(A|C)P(B|C)</code> since <code>P(A, B) is P(A)P(B)</code></li>
</ul>
</li>
<li>Now we can simplify complex things such as <em>full joint distribution using chain rule</em>
<ul>
<li><em>Given A and B are independent</em> <code>P(A, B, C)</code> would <em>normally have 2<sup>3</sup>-1 entries</em></li>
<li>Reduce it by using the above rules + <em>chain rules</em>
<ul>
<li><code>Chain Rule: P(X1, X2, X3, X4) = P(X4 | X1, X2, X3)</code></li>
<li><code>P(A, B, C) = P(C|A, B) P(A, B)</code></li>
<li><code>P(A, B, C) = P(C|A, B) P(B|A) P(A)</code></li>
<li><code>P(A, B, C) = P(C|A) P(B|A) P(A)</code> = 2 + 2 + 1 = 5 entries vs 7</li>
</ul>
</li>
</ul>
</li>
</ul>
